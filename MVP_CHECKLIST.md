# AI 错题本 MVP 开发清单

> 范围：先完成「可靠保存 + 真正按时间复习」这一条最小可用学习闭环；AI 批量能力、PDF 导入、手写判卷留作下一阶段。
> 基线：commit `394d12c`。每一项都应有验收条件与测试。

## MVP-1：安全与数据可靠性

- [~] **M1. API Key 安全存储**：Key 仅存 Keychain/Keystore；旧 SharedPreferences 与 SQLite 的明文 Key 自动迁移并删除。*(两条存储路径均已实现，待设备级测试)*
  - 验收：普通偏好/数据库导出不含 Key；升级用户无需重新填写。
- [~] **M2. 题库切换 SQLite（Drift）**：接入现有 Drift Repository，迁移 SharedPreferences 的题目、练习、复习记录。*(正式 `main.dart` 已使用 Drift；尚需历史 SharedPreferences 迁移与数据保真审计)*
  - 验收：100+ 题保存/检索稳定；迁移幂等；失败不覆盖原始数据。
- [~] **M3. 可恢复备份**：JSON 备份加入 schemaVersion、生成时间、附件清单和导入预览。*(已实现含题图的 Base64 便携式备份、SHA-256 完整性校验、旧格式兼容和 ID 去重；导入预览与大附件压缩待完成)*
  - 验收：旧版文件仍能导入；重复题有跳过/覆盖策略。

## MVP-2：真正的间隔复习（本次首个开发切片）

- [~] **R1. 复习数据模型**：题目持久化 `nextReviewAt`，新题默认立即到期，旧数据兼容为立即到期。*(已实现，待 Flutter 测试验证)*
- [~] **R2. 间隔调度器**：提供「忘记 / 模糊 / 掌握」三档结果，按复习次数计算下次时间。*(已实现，待 Flutter 测试验证)*
- [~] **R3. 到期查询**：首页、复习页、通知只统计 `ready && nextReviewAt <= now` 的题。*(已实现，待 Flutter 测试验证)*
- [~] **R4. 状态更新**：复习操作更新 `lastReviewedAt`、次数、掌握度、下次复习时间并写日志。*(已实现，待 Flutter 测试验证)*
- [~] **R5. 自动化测试**：覆盖新题、旧数据、三档间隔、到期过滤和重置。*(测试已补充；当前环境无 Flutter，未运行)*
- [~] **R6. 复习 UI**：将详情页动作明确为「忘记 / 模糊 / 掌握」，复习页显示下一次计划和今日完成率。*(三档按钮、下次计划、今日完成率已实现；待 Flutter UI 测试验证)*
- [~] **R7. 定时通知**：按用户选定时间排程，仅在存在到期题时提醒，通知深链到复习页。*(已实现权限请求与仅在有到期题时的手动即时提醒；定时排程/深链待实现。iOS 无法在后台重新查询本地题库，计划提醒需配合应用刷新或 Shortcuts 自动化设计)*

## MVP-3：可用性与学习反馈（实现收口，待 Flutter 验收）

- [x] **L1. 错因字段**：概念/审题/计算/方法/格式/粗心，支持编辑与统计。*(6 类结构化错因、详情编辑、首页 Top 3 聚合统计及按错因筛选已完成；待 Flutter UI/回归测试)*
- [x] **L2. 检索升级**：日期、错因、来源、收藏、到期状态筛选及排序。*(来源编辑与动态来源筛选、错因、仅看待复习、仅看收藏、近 7/30 天筛选及最新/最早/下次复习排序已完成；待 Flutter UI/回归测试)*
- [x] **L3. 今日计划**：展示题数、预计耗时、完成率和连续学习。*(首页今日到期题数、3 分钟/题估时、完成率、连续学习与跳转复习已完成；待 Flutter UI/回归测试)*
- [x] **L4. 知识点专项练习**：从薄弱知识点生成/选择练习，并回写表现。*(已聚合知识点关联题的最新练习为跨题合卷；没有可用练习时，通过已配置 AI 生成一组并保存到代表题的下一轮练习，作答仍沿用现有回写链路；待 Flutter/UI/AI 集成验收)*

## MVP-4：AI 可靠性与成本控制

- [ ] **A1. 请求状态机**：取消、超时、离线草稿、局部失败重试。
- [ ] **A2. 结构化输出契约**：版本化 JSON schema、校验与降级解析。
- [ ] **A3. 成本控制**：图片压缩、模型策略、单次估算与预算提醒。
- [ ] **A4. 隐私告知**：首次上传提示服务商、数据处理及原图删除选项。

## 开发顺序

1. 完成 R1–R5 并运行测试；这是可交付的复习内核。
2. 完成 M1，再做 M2（持久化迁移必须独立回归）。
3. 完成 R6–R7，让内核在 UI 与通知中可感知。
4. 用 L1–L4 提升学习价值，最后扩展 AI 与批量能力。
