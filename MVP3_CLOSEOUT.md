# MVP-3 收口说明：可用性与学习反馈

**状态**：实现收口；静态检查通过；等待 Flutter 环境执行分析、单测和真机验收。
**范围原则**：复用现有题库、练习和复习链路，不引入新的数据库 schema 迁移。

## 已交付范围

| 项目 | 交付内容 | 主要入口 |
|---|---|---|
| L1 错因 | 六类错因（概念、审题、计算、方法、格式、粗心）；详情编辑、清除、首页 Top 3 统计、错因筛选 | 题目详情 / 首页 / 错题本 |
| L2 检索 | 学科、知识点、错因、来源、收藏、到期、近 7/30 天筛选；最新/最早/下次复习排序；“全部”完整重置 | 错题本 |
| L3 今日计划 | 今日到期题数、按 3 分钟/题估时、完成进度、连续学习天数、跳转复习 | 首页 |
| L4 专项练习 | 聚合知识点关联题的最新练习为跨题合卷；无可用练习时调用已配置 AI 生成一组并保存到代表题下一轮，作答沿用已有回写链路 | 错题本知识点筛选结果 |

## 数据兼容策略

错因、收藏和来源使用 `tags` 的内部系统标记，避免 Drift schema 升级；它们随现有 SQLite 与 JSON 导入导出链路持久化：

- 错因：`__system_mistake_category:<category>`
- 收藏：`__system_favorite`
- 来源：`__system_source:<source>`

这些标记不会被加入 AI 标签或用户自定义标签的展示/筛选集合。后续需要独立字段时可做无损迁移。

## 手工验收清单

1. 进入任一题目详情：设置、修改、清除错因；回到首页确认统计变化。
2. 收藏一题，重启 App 后确认收藏仍存在；JSON 导出再导入后确认仍存在。
3. 设置来源，确认详情显示来源，错题本出现来源筛选项。
4. 组合“仅看待复习 / 仅看收藏 / 日期范围 / 错因 / 来源”筛选，随后点“全部”，确认列表恢复默认。
5. 连续两天产生复习日志，确认首页连续学习天数；确认今日计划、进度和跳转。
6. 从知识点标签进入筛选结果，使用“专项练习”并完成一道已有练习，确认作答状态保留。
