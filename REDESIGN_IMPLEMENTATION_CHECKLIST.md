# AI错题本界面重设计落地清单

## 交付原则
- 原图始终保留并可查看
- PaddleOCR / MinerU 负责识别与结构化，普通 AI 负责理解与分析
- 识别与 AI 分析可独立完成
- 所有批量/耗时流程可校对、可恢复
- 不凭空补齐题干、公式或 ABCD 选项

## Phase 1：识别工作台（当前优先）
- [x] 上传图片后选择普通 AI / PaddleOCR / MinerU
- [x] PaddleOCR/MinerU 识别后选择：仅保存、逐题选择、全部交给 AI
- [x] 题框、识别文字、公式、表格可校对
- [x] 每题保留 `analyzeWithAi` 决策
- [x] 识别服务状态与失败信息可见
- [ ] 识别结果页显示原图与结构化内容双栏/上下对照
- [x] 识别结果页显示字段状态：题干、公式、选项、图形
- [x] 配置缺失时在选择器内直接显示“去设置”
- [x] 识别任务退出后可恢复

## Phase 2：错题详情工作台
- [x] 顶部摘要：学科、题型、识别来源、AI状态、学习状态
- [x] 原图优先，支持放大、错误提示与附件状态
- [x] OCR/文档内容与 AI 分析分区展示
- [x] 无 AI 分析时提供“交给普通 AI”入口
- [x] 统一识别状态、AI 状态、学习状态标签

## Phase 3：首页与复习闭环
- [x] 今日复习计划首屏
- [x] 待校对/待 AI/识别失败任务
- [x] 薄弱知识点和推荐练习入口
- [x] 复习卡先回忆、后看答案、再评价掌握度

## Phase 4：知识点与推荐
- [ ] 受控知识点树
- [ ] 错因—知识点—题目关联
- [ ] 知识点掌握度
- [ ] 薄弱点优先推荐

## Phase 5：批量工作台与组卷
- [ ] PDF/多图批量识别任务
- [ ] 批量校对与批量 AI 决策
- [ ] 筛选组卷、排序、预览、PDF/打印

## 验收标准
- 用户能明确知道当前使用的识别引擎
- OCR/MinerU 结果可以不调用普通 AI 直接保存
- OCR/MinerU 结果可以全部或逐题交给普通 AI
- 原图、题干、公式、选项、图形不会在详情和导出中丢失
- CI analyze/test/build 全部通过
